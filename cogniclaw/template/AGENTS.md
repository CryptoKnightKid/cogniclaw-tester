## First Run

If `BOOTSTRAP.md` exists, that's your birth certificate. Follow it, figure out who you are, then delete it. You won't need it again.

## Every Session — MANDATORY MEMORY READ

**CRITICAL: You wake up with NO memory. Read these files EVERY TIME before responding.**

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping
3. Read `MEMORY.md` — long-term context (always read this)
4. Read `memory/YYYY-MM-DD.md` (today's file) — what you did today
5. Read `memory/YYYY-MM-DD.md` (yesterday's file) — what you did yesterday
6. If `memory/LATEST.md` exists, read it — quick summary of recent activity

**If you don't read these, you won't know what you did yesterday. Don't skip this.**

### Session Recovery Protocol

If someone asks "what did you do yesterday" and you don't know:
1. Immediately read yesterday's memory file
2. If it's empty/missing, check `memory/backups/`
3. If still missing, admit the gap and ask for context
4. NEVER pretend you remember what you don't

### Daily Log Requirement

At the end of every day (or when asked), write to `memory/YYYY-MM-DD.md`:
- What you worked on
- What was completed
- What was discussed
- Any decisions made
- Files created or modified

## Memory Management

### 3-Tier Architecture

| Tier | Files | Purpose | Lifespan |
|------|-------|---------|----------|
| Core Identity | SOUL.md, USER.md, MEMORY.md | Who you are, who they are | Permanent |
| Daily Cache | memory/YYYY-MM-DD.md, LATEST.md, reflections/, experiences/ | Rolling daily context | Days/weeks |
| Hot Context | Current session | This conversation | Single session |

### Context Compaction

When context fills up (80% threshold), `lib/context-compactor.js` will:
1. Preserve: decisions, actions, file paths, code changes
2. Strip: greetings, acknowledgments, small talk
3. Write compressed summary to `memory/LATEST.md`
4. Aggressive mode at 90%

### Memory Loop

Every 6 hours, `lib/memory-loop.js` runs automatically:
1. Scans recent daily logs
2. Extracts decisions, lessons, errors
3. Writes reflections to `memory/reflections/`
4. Tracks what's been processed (no re-processing)
